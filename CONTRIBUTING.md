# Contributing to Keyscope

Thank you for your interest in contributing to Keyscope! We welcome contributions from the community.

## Getting Started

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally.
3.  Run `flutter pub get` to install dependencies.
4.  Create a new branch for your feature or bug fix.

## Code Style

* We follow the official [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style).
* Run `dart format .` before committing.
* Ensure zero warnings in `flutter analyze`.

## Pull Request Process

1.  Push your changes to your fork.
2.  Submit a Pull Request (PR) to the `main` branch.
3.  Provide a clear description of the changes and link any relevant issues.
